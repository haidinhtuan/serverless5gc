## Quick Recreation (TL;DR)

```bash
# 1. Build function images (~30 min first time)
bash deploy/openfaas/build-functions.sh

# 2. Provision 5 VMs (~15 min)
bash deploy/ionos/provision.sh
source deploy/ionos/vm-ips.env

# 3. Setup each VM (can run in parallel after step 2)
scp deploy/ionos/setup-serverless.sh root@$SERVERLESS_IP:/tmp/ && \
  ssh root@$SERVERLESS_IP 'bash /tmp/setup-serverless.sh'

scp -r deploy/baselines/open5gs/ root@$OPEN5GS_IP:/opt/open5gs/ && \
  ssh root@$OPEN5GS_IP 'bash -s' < deploy/ionos/setup-open5gs.sh

scp -r deploy/baselines/free5gc/ root@$FREE5GC_IP:/opt/free5gc/ && \
  ssh root@$FREE5GC_IP 'bash -s' < deploy/ionos/setup-free5gc.sh

ssh root@$LOADGEN_IP 'bash -s' < deploy/ionos/setup-loadgen.sh

ssh root@$MONITORING_IP "SERVERLESS_IP=$SERVERLESS_IP OPEN5GS_IP=$OPEN5GS_IP \
  FREE5GC_IP=$FREE5GC_IP LOADGEN_IP=$LOADGEN_IP MONITORING_IP=$MONITORING_IP \
  bash -s" < deploy/ionos/setup-monitoring.sh

# 4. Deploy OpenFaaS functions (~10 min)
bash deploy/openfaas/deploy-functions.sh $SERVERLESS_IP

# 5. Provision subscribers (serverless target only)
# Python script via OpenFaaS gateway - 1000 subscribers

# 6. Run evaluation
bash deploy/ionos/run-eval.sh
```

**Total setup time**: ~45-60 minutes (most VMs can be set up in parallel).

- **ionosctl** v6.9.6+ installed (`~/.config/ionosctl/config.yaml` for auth)

## 3. Provisioning Scripts

### 3.1 provision.sh

**Location**: `deploy/ionos/provision.sh`

Creates the full datacenter from scratch:
1. Creates datacenter named `s5gc-eval-YYYYMMDD-HHMMSS` in Frankfurt (de/fra)
2. Creates public LAN with DHCP
3. Creates 5 VMs with Ubuntu 24.04 boot volumes
4. Waits for DHCP IP assignment (~45s + retries)
5. Tests SSH connectivity (up to 15 min timeout)
6. Outputs `deploy/ionos/vm-ips.env`

**VM definitions**:
```bash
VM_DEFS=(
    "serverless5gc:4:8192:50"
    "open5gs:4:8192:50"
    "free5gc:4:8192:50"
    "loadgen:4:8192:40"
    "monitoring:4:8192:40"
)
```

### 3.2 setup-serverless.sh

**Target**: VM1 (serverless5gc)

Installs (in order):
1. K3s single-node (`--disable traefik`)
2. Helm
3. OpenFaaS (Helm chart, timeouts=120s, basic_auth=false, queue_worker_max_inflight=50)
4. faas-cli
5. Redis pod (`redis:7-alpine`, PVC 1Gi, NodePort 32660, appendonly=yes, maxmemory=256mb)
6. etcd pod (`quay.io/coreos/etcd:v3.5.17`, PVC 1Gi)
7. cAdvisor DaemonSet (`gcr.io/cadvisor/cadvisor:v0.49.1`, port 8081)
8. node-exporter DaemonSet (`prom/node-exporter:v1.7.0`, port 9100)

**SCTP proxy** is deployed separately after functions:
```bash
# Binary at /usr/local/bin/sctp-proxy
OPENFAAS_GATEWAY='http://localhost:31113/function/' \
REDIS_ADDR='localhost:32660' \
nohup /usr/local/bin/sctp-proxy > /var/log/sctp-proxy.log 2>&1 &
```

### 3.3 setup-open5gs.sh

**Target**: VM2 (open5gs)

1. Install Docker
2. Copy configs: `scp -r deploy/baselines/open5gs/ root@$OPEN5GS_IP:/opt/open5gs/`
3. `docker compose up -d` (12 services on network 10.33.33.0/24)
4. cAdvisor + node-exporter

**Image**: `gradiant/open5gs:2.7.0`

### 3.4 setup-free5gc.sh

**Target**: VM3 (free5gc)

1. Install Docker
2. Build gtp5g kernel module (clone from GitHub, make, make install, modprobe)
3. Copy configs: `scp -r deploy/baselines/free5gc/ root@$FREE5GC_IP:/opt/free5gc/`
4. `docker compose up -d` (11 services + webui on network 10.100.200.0/24)
5. cAdvisor + node-exporter

**Image**: `free5gc/nrf:v4.2.0` etc.

### 3.5 setup-loadgen.sh

**Target**: VM4 (loadgen)

1. Install build-essential, cmake, git, libsctp-dev, lksctp-tools
2. Upgrade CMake if < 3.17.0
3. Clone + build UERANSIM v3.2.6 at `/opt/UERANSIM`
4. Install gNB/UE config templates to `/opt/ueransim-configs/`
5. Docker + node-exporter

### 3.6 setup-monitoring.sh

**Target**: VM5 (monitoring)

**Requires env vars**: `SERVERLESS_IP`, `OPEN5GS_IP`, `FREE5GC_IP`, `LOADGEN_IP`, `MONITORING_IP`

1. Install Docker
2. Generate Prometheus config (scrape_interval=5s, 30-day retention)
3. Docker Compose: Prometheus v2.51.0, Grafana v10.4.1, cost-exporter, node-exporter
4. Prometheus targets: OpenFaaS gateway, cAdvisor on 3 VMs, node-exporter on all VMs

## 5. Subscriber Provisioning (Serverless Only)

**1000 subscribers** pre-provisioned in Redis.

| Field | Value |
|-------|-------|
| IMSI range | `imsi-001010000000001` to `imsi-001010000001000` |
| Key (K) | `465B5CE8B199B49FAA5F0A2EE238A6BC` |
| OPC | `E8ED289DEBA952E4283B54E88E6183CA` |
| AMF | `8000` |
| SQN | `000000000020` (hex, = 32 decimal, **must be >= 32**) |
| PLMN | 001/01 |
| S-NSSAI | sst=1, sd=010203 |
| DNN | internet |

**Storage**: Redis key `subscribers/imsi-001010NNNNNNNNN`

**Method**: POST to `udr-data-write` function with JSON body (base64-encoded auth fields).

**CRITICAL**: Redis FLUSHALL destroys all subscriber data. Never flush Redis between runs. Only restart the sctp-proxy process.

## 8. Critical Gotchas

### Redis & Subscribers
- **NEVER** `FLUSHALL` Redis — destroys 1000 pre-provisioned subscribers
- SQN must be >= 32 for UERANSIM authentication
- Redis NodePort: **32660** (not 6379 from host)

### SCTP Proxy
- Direct binary at `/usr/local/bin/sctp-proxy`, **NOT** systemd, **NOT** K3s pod
- `systemctl restart sctp-proxy` fails silently — kill & restart manually
- Uses `localhost:31113` for gateway (K3s DNS doesn't resolve from host)
- Uses `localhost:32660` for Redis (NodePort)

### OpenFaaS
- Gateway NodePort: **31113**
- Timeouts: 120s (for long registration/PDU scenarios)
- basic_auth: **disabled**

### etcd
- Image: `quay.io/coreos/etcd:v3.5.17` (**NOT bitnami** — unavailable)

### free5GC
- Requires **gtp5g kernel module** compiled from source (UPF dependency)
- Must include `cert/` subdirectory in configs

### Baselines
- `docker compose restart` between runs to clear accumulated UE state
- Without restart: T3580 retry floods from duplicate IMSIs

### UERANSIM
- Config templates use `__LOADGEN_IP__` and `__TARGET_AMF_IP__` placeholders
- Validates SQN >= 32

### IONOS
- Auth: config file (`~/.config/ionosctl/config.yaml`), NOT env var
- Scripts explicitly `unset IONOS_TOKEN`
- SSH key: `~/.ssh/id_rsa` injected into VMs
